# Resume Skill

`resume-skill` 是一个帮你写好中文简历的 AI skill。它最大的特点是：既面向 GEO/AI 搜索与招聘筛选，又能适配不同专业、不同岗位方向的简历写作。

你可以把它交给 Claude Code、Codex 或兼容 skill 的 AI 助手使用，让 AI 从 HR、ATS、招聘 Agent 和生成式搜索的角度，帮你诊断、改写、生成一份更适合投递和被发现的简历。

它适合不懂简历排版、不确定怎么写项目经历、担心简历过不了 HR 或 AI 初筛的人使用。

## 核心特点

- GEO 友好：让简历更容易被 AI 搜索、招聘 Agent、智能问答和简历筛选系统正确理解。
- 多专业适配：不只适合程序员，也适合产品、运营、市场、设计、财会、金融、法学、教育、医学、土木、机械、新媒体、行政等方向。
- 岗位导向：同一段经历会按目标岗位重写，不同专业背景也能转成招聘方看得懂的岗位语言。
- HR 可读：减少空话和模板化表达，让 HR 更快看清你的定位、能力和证据。
- HTML/PDF 友好：保留可复制、可读取的文本内容，兼顾手机阅读、电脑阅读和 PDF 投递。

## 适用人群

- 第一次写简历，不知道应该放什么、不该放什么。
- 已有简历，但投递效果一般，想让 AI 帮你从 HR 视角检查问题。
- 来自不同专业，想把课程、实习、项目、竞赛、科研或作品集转成简历语言。
- 准备转岗、转行、找实习、找全职，想把经历改写成更匹配目标岗位的版本。
- 手里有岗位 JD，想让简历更贴近招聘要求和常见筛选关键词。
- 希望简历对 AI 搜索、招聘 Agent、ATS 和 GEO 更友好。
- 想生成一份手机、电脑和 PDF 都能正常阅读的 HTML 简历。
- 做个人求职材料整理，希望先有一个可信、清晰、可继续打磨的版本。

## 快速入门

### 1. 安装 skill

如果你已经把本目录安装到 Claude Code 或 Codex 的 skills 目录，可以跳过这一步。

Claude Code:

```bash
mkdir -p ~/.claude/skills
ln -s /Users/blingabc/PycharmProjects/resume-skill ~/.claude/skills/resume-skill
```

Codex:

```bash
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
ln -s /Users/blingabc/PycharmProjects/resume-skill "${CODEX_HOME:-$HOME/.codex}/skills/resume-skill"
```

如果你的 Codex 环境使用 `~/.agents/skills`：

```bash
mkdir -p ~/.agents/skills
ln -s /Users/blingabc/PycharmProjects/resume-skill ~/.agents/skills/resume-skill
```

### 2. 准备材料

你不需要先整理得很完美。能提供多少就提供多少，AI 会根据缺失信息继续追问。

建议准备：

- 现有简历，或一段原始经历描述。
- 目标岗位，例如“产品运营”“数据分析师”“Java 后端开发”。
- 目标岗位 JD，越具体越好。
- 工作、实习、项目、教育、证书、技能、作品链接等材料。
- 你希望投递的城市、行业、公司类型或薪资方向。

### 3. 直接这样问 AI

改写已有简历：

```text
用 $resume-skill 帮我根据这份原始简历，改写成适合产品运营岗位的版本。请先指出主要问题，再给出改写后的简历。
```

从零生成简历：

```text
用 $resume-skill 根据我的经历新建一份适合数据分析岗位的中文简历，并生成 HTML 版本。
```

不同专业适配：

```text
用 $resume-skill 把我的市场营销专业经历，改写成适合用户运营岗位投递的简历。
```

按 JD 优化简历：

```text
用 $resume-skill 对照下面这份岗位 JD，帮我优化简历，让内容更适合 HR 和 AI 初筛。
```

GEO/AI 搜索优化：

```text
用 $resume-skill 帮我优化这份简历，让它更适合 ATS、招聘 Agent 和 GEO 检索，但不要堆砌关键词。
```

只做专业审阅：

```text
用 $resume-skill 从 HR 角度审阅我的简历，指出最影响投递结果的问题，并给出修改建议。
```

导出 PDF：

```text
用 $resume-skill 生成 HTML 简历，并帮我导出 PDF。
```

## 功能

- 简历体检：找出定位不清、经历空泛、关键词不足、表达不可信等问题。
- 简历改写：把“负责、参与、熟悉”这类弱表达，改成更像真实工作成果的表达。
- 从零生成：根据你的经历素材，整理成完整的中文简历。
- 多专业适配：把不同专业里的课程、项目、论文、竞赛、实习、作品集转成岗位相关表达。
- 岗位匹配：围绕目标岗位或 JD 调整标题、摘要、经历重点和技能关键词。
- HR 视角建议：告诉你哪些内容会被 HR 质疑，哪些内容最应该补充证据。
- GEO/ATS/AI 筛选友好：让简历保留可读取文本，关键词自然出现，避免堆砌和隐藏关键词。
- HTML 简历：生成适合手机、电脑和 PDF 打印的简历页面。
- PDF 导出：需要时把 HTML 简历导出为 PDF。
- 内容质检：检查简历里是否还有占位符、过长要点、弱表达、隐藏内容等问题。

## 工作原理

`resume-skill` 不是简单地把文字润色得更好看，而是按招聘筛选的过程来处理简历。

1. 先判断你要投什么岗位，以及你当前材料里最有价值的证据。
2. 再识别你的专业背景：把课程、项目、论文、竞赛、实习、作品集等材料，转成目标岗位能理解的能力证据。
3. 再从 HR 视角检查简历：定位是否清楚、经历是否可信、成果是否具体、风险点是否明显。
4. 然后按目标岗位重写内容：突出更相关的经历，补充关键词，但不做夸大和造假。
5. 如果需要 GEO/AI 优化，会补齐标题、摘要、关键词、结构化信息和可读取文本，让招聘 Agent 和生成式搜索更容易理解。
6. 如果需要 HTML/PDF，会使用内置模板生成可阅读、可复制、可被筛选系统理解的简历。
7. 最后做基础质检，尽量避免占位内容、隐藏关键词、格式问题和明显弱表达。

简单说，它的目标是让简历同时服务三类读者：

- HR：快速看懂你适合什么岗位。
- 面试官：看到真实经历和可追问的证据。
- ATS/招聘 Agent/AI 搜索：能正确读取你的信息、关键词、专业背景和经历结构。

## 使用建议

- 一份简历优先服务一个目标岗位。不同岗位最好生成不同版本。
- 不要让 AI 编造公司、学校、职级、证书、项目或数字。
- 如果没有准确数据，可以先写规模、频率、对象、时长、协作人数等事实。
- 如果 AI 问你补充信息，优先补“结果、数据、职责边界、项目背景”。
- 最终简历里的事实需要你自己确认，尤其是时间、公司、学历、证书和联系方式。

## 维护者说明

普通使用者不需要阅读本节。只有在你要维护模板、脚本或测试时才需要这些命令。

主要文件：

- `SKILL.md`：AI 助手会读取的核心工作说明。
- `references/`：HR 审阅、岗位写法、ATS/AI 友好简历的参考资料。
- `assets/resume-template.html`：正式 HTML 简历模板。
- `resume-template.html`：便于本地预览的模板副本。
- `scripts/`：HTML 质检和 PDF 导出脚本。
- `tests/`：脚本测试。

检查内置模板：

```bash
python3 scripts/check_resume_html.py assets/resume-template.html --allow-placeholders
```

检查正式用户简历：

```bash
python3 scripts/check_resume_html.py path/to/resume.html
```

导出 PDF：

```bash
python3 scripts/export_resume_pdf.py path/to/resume.html path/to/resume.pdf
```

运行测试：

```bash
python3 -m unittest tests/test_check_resume_html.py
```
