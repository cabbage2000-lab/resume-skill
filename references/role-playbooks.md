# Role-Specific Resume Playbooks

Use this reference when the user names a target role, provides a JD, or asks for role-specific optimization.

## Product Operations / 产品运营

Best evidence:

- Funnel, conversion, retention, activation, recall, content, campaign, community, user feedback, or lifecycle work.
- Data analysis tied to action: segmentation, channel comparison, cohort, A/B review, dashboard, weekly review.
- Cross-functional delivery with product, design, content, sales, or customer support.

Keyword variants:

- 产品运营 / 用户增长 / 活动运营 / 内容运营 / 数据复盘 / 用户反馈 / 转化漏斗 / SOP / 项目推进

Common HR concerns:

- Only says "运营" without proving business outcome.
- Lists tools but does not show how analysis changed decisions.
- Uses campaign result numbers without clarifying candidate contribution.

Recommended structure:

- Summary, core skills, experience, selected projects, education, awards/certifications.

## Data Analyst / 数据分析

Best evidence:

- SQL, Python, Excel, BI dashboards, metric systems, data cleaning, cohort analysis, funnel analysis, experiment review.
- Decisions influenced by analysis: product change, campaign allocation, inventory planning, operational efficiency.
- Data scale, refresh frequency, stakeholder group, and recurring reporting cadence.

Keyword variants:

- 数据分析 / SQL / Python / Excel / BI / dashboard / funnel analysis / cohort analysis / 指标体系 / 数据清洗

Common HR concerns:

- Charts are described without business context.
- Tool list is stronger than project evidence.
- Data scale, quality checks, and final adoption are missing.

Recommended structure:

- Summary, technical skills, work or project experience, education, portfolio links, certifications.

## Software Engineer / 技术开发

Best evidence:

- Production features, architecture decisions, performance, reliability, testing, deployment, API, database, observability.
- Clear ownership boundaries: module owner, feature owner, bug fix owner, reviewer, maintainer.
- Technical impact: latency, throughput, stability, test coverage, incident reduction, developer efficiency.

Keyword variants:

- 后端开发 / 前端开发 / full-stack / API / database / CI/CD / testing / performance / reliability / system design

Common HR concerns:

- Project descriptions read like course assignments.
- Tech stack appears without implementation depth.
- Claims of ownership are too broad for candidate seniority.

Recommended structure:

- Summary, technical skills, work experience, projects, open-source or portfolio, education.

## Technical Manager / 技术经理

Best evidence:

- Team size, reporting line, delivery scope, roadmap ownership, project portfolio, hiring, onboarding, mentoring, review, or performance feedback.
- Engineering management tied to outcomes: delivery predictability, quality, stability, incident reduction, release cadence, developer efficiency, cross-team alignment.
- Technical depth that supports management: architecture review, technical planning, risk control, code review standards, engineering process, observability, quality gates.

Keyword variants:

- 技术经理 / 技术负责人 / engineering manager / tech lead manager / 团队管理 / 项目交付 / 技术规划 / 架构评审 / 研发效能 / 质量治理

Common HR concerns:

- Only says "带团队" without team size, scope, or measurable delivery result.
- Leadership claims are broad, but examples show only individual contributor work.
- Management process is listed as meetings and coordination, without quality, efficiency, retention, or business impact.
- Technical depth disappears after moving into management, making seniority hard to judge.

Recommended structure:

- Summary, leadership scope, technical strengths, management and delivery experience, selected projects, team/process improvements, education/certifications.

## System Architect / 系统架构师

Best evidence:

- Architecture design across services, domains, data flows, integrations, cloud infrastructure, security, reliability, scalability, or migration programs.
- Clear decision records: constraints, trade-offs, alternatives evaluated, standards created, review mechanism, rollout path, and measurable impact.
- Cross-team technical influence: architecture governance, API contracts, platform capabilities, performance tuning, cost optimization, incident prevention, technical debt reduction.

Keyword variants:

- 系统架构师 / 架构设计 / distributed systems / microservices / cloud-native / high availability / scalability / DDD / API gateway / architecture governance

Common HR concerns:

- Architecture is described as diagrams or buzzwords without constraints, trade-offs, or adoption evidence.
- Claims are too broad for the candidate's actual decision authority.
- Technology choices are listed without explaining business context, system scale, reliability target, or migration risk.
- Resume does not separate hands-on implementation, architecture review, and organization-level technical influence.

Recommended structure:

- Summary, architecture domains, core technical skills, architecture and platform experience, selected architecture projects, standards/governance work, education/certifications.

## Marketing Growth / 市场增长

Best evidence:

- Channel strategy, campaign planning, content, paid/organic acquisition, leads, conversion, brand assets, CRM, lifecycle.
- Budget, audience size, channel mix, conversion rate, cost per lead, retention, campaign cadence.
- Collaboration with sales, product, design, agency, KOL, or community partners.

Keyword variants:

- 市场增长 / 品牌传播 / 内容营销 / 渠道投放 / 线索转化 / CRM / campaign / lead generation / conversion

Common HR concerns:

- Uses buzzwords without campaign scale.
- Mentions exposure but not conversion or follow-up.
- Overstates strategic ownership for execution-only work.

Recommended structure:

- Summary, experience, campaigns/projects, data/tools, education, awards.

## Fresh Graduate / 应届生

Best evidence:

- Internships, projects, campus leadership, competitions, coursework only when directly relevant.
- Proof of execution discipline: timeline, stakeholder count, output, repeated cadence, result.
- Transferable skills framed toward the target role.

Keyword variants:

- 实习经历 / 项目经历 / 校园经历 / 数据分析 / 项目推进 / 沟通协作 / 学习能力 / 目标岗位关键词

Common HR concerns:

- Education takes too much space when experience is stronger.
- Campus titles are listed without deliverables.
- Self-evaluation replaces proof.

Recommended structure:

- Profile, education, internships, projects, campus leadership, skills, awards/certifications.

## Career Transition / 转行候选人

Best evidence:

- Transferable responsibilities, recent role-specific projects, certifications, portfolio, volunteer or freelance work.
- Clear reason for target direction, shown through actions rather than a long narrative.
- Evidence that reduces ramp-up concern.

Keyword variants:

- transferable skills / portfolio / certification / domain knowledge / stakeholder management / analysis / delivery

Common HR concerns:

- Old career dominates the resume.
- Target role appears only in the summary.
- Training certificates are present but applied projects are weak.

Recommended structure:

- Summary, transferable skills, relevant projects, prior experience, education/certifications.
